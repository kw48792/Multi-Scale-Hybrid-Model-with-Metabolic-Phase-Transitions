# Contributing Guidelines

Thank you for your interest in this research code repository!

## Purpose

This repository primarily serves to make the research from the paper "Multi-Scale Hybrid Modeling to Predict Cell Culture Process with Metabolic Phase Transitions" reproducible and accessible.

## Types of Contributions

We welcome the following types of contributions:

### 1. Bug Reports
If you find a bug in the code:
- Check if it's already reported in Issues
- Create a new issue with:
  - Description of the bug
  - Steps to reproduce
  - Expected vs. actual behavior
  - Your environment (OS, Python version, package versions)

### 2. Bug Fixes
If you can fix a bug:
- Fork the repository
- Create a branch for your fix
- Make minimal changes to fix the issue
- Test your fix
- Submit a pull request with a clear description

### 3. Documentation Improvements
- Clarifications to README
- Additional examples
- Corrections to docstrings
- Improved explanations

### 4. Extensions and Enhancements
If you extend the model or add new features:
- Ensure they don't break existing functionality
- Add tests and examples
- Update documentation
- Explain the scientific basis for changes

## Code Standards

### Python Style
- Follow PEP 8
- Use meaningful variable names
- Add docstrings to functions and classes
- Keep functions focused and modular

### Documentation
- Update README if adding new features
- Add examples for new functionality
- Document any new dependencies

### Testing
- Ensure existing examples still work
- Add examples for new features
- Verify reproducibility

## Pull Request Process

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Make your changes
4. Test your changes
5. Commit with clear messages
6. Push to your fork
7. Open a pull request with:
   - Clear description of changes
   - Motivation for the changes
   - Any relevant issue numbers

## Questions?

For questions about the research or code:
- Open an issue on GitHub
- Check existing documentation
- Contact the authors

## Code of Conduct

### Our Standards

- Be respectful and professional
- Focus on scientific accuracy
- Assume good intentions
- Provide constructive feedback

### Scientific Integrity

- Maintain reproducibility
- Document all changes
- Cite sources appropriately
- Don't plagiarize code or data

---

Thank you for helping make this research more accessible and reproducible!
